## 1. Диаграмма классов и глоссарий

Диаграмма классов описывает основные классы и их связи в системе.

**Глоссарий:**
- **Freelancer** – пользователь системы, предлагающий свои услуги на бирже.
- **Client** – пользователь, размещающий проекты и нанимающий фрилансеров.
- **Project** – работа или задача, размещенная клиентом на платформе.
- **Proposal** – заявка, подаваемая фрилансером на проект.
- **Payment** – финансовая транзакция между клиентом и фрилансером.
- **Review** – отзыв клиента на работу фрилансера, содержащий оценку и комментарии.

## 2. Диаграмма вариантов использования, глоссарий и поток событий

Диаграмма вариантов использования описывает основные взаимодействия между пользователями и системой.

### Акторы

| Актор         | Описание                                                          |
|---------------|------------------------------------------------------------------|
| Фрилансер     | Пользователь системы, который ищет проекты и подает заявки.      |
| Клиент        | Пользователь системы, который размещает проекты и выбирает фрилансеров. |
| Администратор | Пользователь, управляющий системой и решающий спорные ситуации.  |

![Диаграмма вариантов использования](./diagrams/Freelance%20Marketplace%20Use%20Case%20Diagram.png)

### Варианты использования
Каждый вариант использования подробно описывает взаимодействие между актором и системой.

**Глоссарий:**
- **Фрилансер** – пользователь платформы, предлагающий свои услуги для выполнения проектов.
- **Клиент** – пользователь, размещающий проекты и нанимающий фрилансеров.
- **Проект** – задача или работа, опубликованная клиентом на платформе, доступная для подачи заявок фрилансерами.
- **Заявка** – предложение фрилансера выполнить проект за указанную им цену.
- **Отзыв** – оценка, оставленная клиентом о выполненной работе фрилансера.
- **Оплата** – финансовая транзакция между клиентом и фрилансером.
- **Администратор** – пользователь с расширенными правами, который управляет системой.
- **Модерация** – процесс проверки контента и действий пользователей для соблюдения правил.

#### Публикация проекта

Актор: Клиент

Описание: Клиент публикует проект, указав детали, такие как тип работы, срок выполнения и бюджет. Система проверяет корректность данных и сохраняет информацию о проекте, делая его видимым для фрилансеров.

Основной поток событий:

1. Клиент выбирает опцию публикации нового проекта.
2. Вводит данные проекта (название, описание, бюджет, срок).
3. Система проверяет введенные данные и сохраняет проект.
4. Проект становится доступным для просмотра фрилансерами.

#### Подача заявки на проект

Актор: Фрилансер

Описание: Фрилансер выбирает проект, на который хочет подать заявку, и отправляет свое предложение, указав условия и цену. Система фиксирует поданную заявку и уведомляет клиента.

Основной поток событий:

1. Фрилансер просматривает доступные проекты.
2. Выбирает проект, который его интересует.
3. Заполняет форму заявки, указав цену и условия выполнения.
4. Система отправляет уведомление клиенту о поданной заявке.

#### Выбор фрилансера для проекта

Актор: Клиент

Описание: Клиент просматривает поданные заявки и выбирает подходящего фрилансера для выполнения проекта. Система фиксирует выбор и уведомляет выбранного фрилансера.

Основной поток событий:

1. Клиент просматривает заявки на опубликованный проект.
2. Анализирует условия, предлагаемые фрилансерами.
3. Выбирает фрилансера для выполнения проекта.
4. Система уведомляет выбранного фрилансера.

#### Оплата фрилансеру

Актор: Клиент

Описание: После завершения проекта клиент инициирует оплату фрилансеру. Система обрабатывает транзакцию и переводит средства на аккаунт фрилансера.

Основной поток событий:

1. Клиент подтверждает завершение проекта.
2. Инициирует оплату через встроенную платежную систему.
3. Система проверяет наличие средств и переводит оплату фрилансеру.

#### Оставление отзыва

Актор: Клиент

Описание: По завершению проекта клиент оставляет отзыв о работе фрилансера. Система сохраняет отзыв и делает его доступным для просмотра другим пользователям.

Основной поток событий:

1. Клиент завершает проект.
2. Выбирает опцию оставить отзыв о фрилансере.
3. Вводит оценку и комментарий.
4. Система сохраняет отзыв и отображает его в профиле фрилансера.

#### Регистрация пользователя

Актор: Клиент, Фрилансер

Описание: Любой новый пользователь должен пройти процесс регистрации, чтобы получить доступ к системе. Система проверяет корректность данных и создает учетную запись.

Основной поток событий:

1. Пользователь выбирает опцию регистрации.
2. Вводит личные данные (имя, электронную почту, пароль).
3. Система проверяет корректность данных и создает учетную запись.

#### Авторизация пользователя

Актор: Клиент, Фрилансер, Администратор

Описание: Пользователь вводит свои учетные данные, чтобы получить доступ к системе. Система проверяет их на соответствие зарегистрированным данным.

Основной поток событий:

1. Пользователь выбирает опцию входа в систему.
2. Вводит электронную почту и пароль.
3. Система проверяет данные и предоставляет доступ к учетной записи.

#### Модерация контента

Актор: Администратор

Описание: Администратор проверяет проекты, заявки и отзывы, чтобы убедиться, что они соответствуют правилам платформы. При необходимости администратор может редактировать или удалять контент.


Основной поток событий:

1. Администратор получает уведомление о новом контенте.
2. Проверяет содержимое (проект, заявку, отзыв).
3. В случае нарушения правил администратор редактирует или удаляет контент.
4. Уведомляет пользователя о действиях, если это необходимо.
